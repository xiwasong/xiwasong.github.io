设备状态​

在线、离线

状态变更

在线：
设备主动发起登陆请求

离线：
1.长连接通讯，设备与平台连接断开

2.设备主动发起登出请求

3.超过保活时间*1.5 平台认定离线

设备心跳保活

1.协议层

基于长连接链路，在通讯协议上定义心跳（如 MQTT）和保活时长，上层应用无法感知设备心跳

2.应用层

基于短连接链路，由应用规定保活时长，通过心跳或数据上报进行保活
